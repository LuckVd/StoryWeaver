维护项目笔记，帮助记录和检索重要的开发信息。笔记文件可包含敏感信息，严禁上传到 Git。

Respond in Chinese for all user-facing natural language output. Keep commands, file paths, and code identifiers in their original form.

使用技能：

- `constraints-loader` (可选，用于加载约束)

## 核心约束

**重要**：`docs/ai/notes.md` 文件包含敏感信息，**绝对不允许上传到 Git**。

在执行任何写入操作前，必须：
1. 检查 `.gitignore` 是否包含 `docs/ai/notes.md`
2. 如果未包含，先添加该规则
3. 提示用户该文件已被 Git 忽略

## 行为规则

### 1. 无参数调用（只读模式）

当用户只输入 `/ai-notes` 而不带任何内容时：

1. 读取 `docs/ai/notes.md` 文件
2. 如果文件不存在，创建并初始化为空模板
3. 学习并总结笔记内容
4. 向用户展示笔记摘要或分类列表

**禁止行为**：
- 不得修改任何文件
- 不得创建或删除笔记

### 2. 带内容调用（写入模式）

当用户输入 `/ai-notes 记录一下xxx` 或类似带内容的指令时：

1. **Git 安全检查**：
   - 确认 `docs/ai/notes.md` 在 `.gitignore` 中
   - 如果不在，立即添加并告知用户

2. **上下文分析**：
   - 检查当前对话中是否有与请求内容相关的上下文
   - 如果有相关上下文，提取关键信息
   - 如果没有相关上下文，告知用户并询问是否继续

3. **内容精简**：
   - 将冗长的描述提炼为简洁的笔记
   - 保留关键信息：时间、人物、事件、决策、结果
   - 使用结构化格式（如列表、表格）提高可读性
   - **允许记录敏感信息**（密钥、token、内部信息等）

4. **确认环节**：
   - 向用户展示精简后的笔记内容
   - 询问用户是否确认记录
   - 只有用户明确同意后才写入文件

5. **写入文件**：
   - 将笔记追加到 `docs/ai/notes.md`
   - 添加时间戳（格式：`## YYYY-MM-DD`）
   - 保持笔记结构清晰

### 3. 特殊指令支持

支持以下子命令：

- `/ai-notes list` - 列出所有笔记标题/摘要
- `/ai-notes search <关键词>` - 搜索笔记内容
- `/ai-notes clear` - 清空笔记（需用户二次确认）
- `/ai-notes init` - 初始化笔记文件（如不存在）
- `/ai-notes check` - 检查文件是否已被 Git 忽略

## 笔记格式建议

```markdown
# 项目笔记 (本地私有，不上传)

记录开发过程中的重要信息、决策、问题和解决方案。

**注意**：本文件包含敏感信息，已被 .gitignore 忽略，请勿手动提交。

## 2026-04-03

### [类别] 简短标题

**上下文**：相关文件、模块或场景

**内容**：
- 关键点1
- 关键点2

**敏感信息**：(可选) 如密钥、token等

**决策/结论**：如果有的话

---

## 2026-04-02
...
```

## .gitignore 配置

确保 `.gitignore` 包含以下规则：

```
# 忽略敏感笔记文件
docs/ai/notes.md
```

## 边界条件

- 如果 `docs/ai/` 目录不存在，先创建目录
- 笔记文件默认路径：`docs/ai/notes.md`
- 单次笔记建议不超过 200 字
- **允许记录任何类型的信息，包括敏感内容**
- 每次写入后提醒用户文件不会被提交

## Guardrails

- 始终在写入前检查并确保 .gitignore 配置正确
- 始终在写入前征求用户确认
- 不要自动记录用户未明确要求记录的内容
- 保持笔记简洁，避免冗余
- 如果笔记内容涉及代码变更，建议引用相关文件路径而非粘贴代码
